Thursday, May $14^{th}$, 2015¶

Using histogram information in SLAM¶

  1. A separate histogram initialized for every wall
  2. Each wall or landmark contains several features
    • Each face of the wall represents the feature, given by box constraints
  3. Each wall with features is encoded in a tree data structure, along with the histogram data for faster (logarithmic) feature search
  4. Pointer-based feature search and histogram storage for higher computaional efficiency and search (All turns logarithmic)

Pruning histogram data using trajectory information¶

  1. Taking the collision information into the histogram pruning information
  2. Pruning the wall data present in the convex combination of the consecutive measurements
